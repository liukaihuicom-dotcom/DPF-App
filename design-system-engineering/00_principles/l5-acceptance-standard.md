# L5 Acceptance Standard

The design-system engineering package is production ready only when these gates pass:

- Tokens have schema, modes, aliases, export mapping, QA rules, and code mapping.
- Components have manifest entries, token bindings, states, accessibility notes, and forbidden usage.
- Business components include domain, state, permission, risk, data input, platform, and QA ownership.
- Icons are registered with source, license, semantics, state, size, and runtime mapping.
- Patterns are machine-readable and mapped to platform rules.
- AI runtime has a readable index and hard rules for missing inputs.
- Release records include changelog, migration, deprecated assets, impact, and checklist.
