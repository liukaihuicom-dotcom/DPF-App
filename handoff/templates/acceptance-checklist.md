# Acceptance Checklist

| Gate | Requirement | Status |
|---|---|---|
| Business | Module, role, goal, and rules are clear | [ ] |
| Permission | Page, button, and field permissions are clear | [ ] |
| State | All required states are covered | [ ] |
| Design System | Tokens and shared components are used | [ ] |
| Component | Component APIs are documented | [ ] |
| Interaction | Triggers, feedback, and exceptions are complete | [ ] |
| Field | Field validation and API mapping are complete | [ ] |
| API | Request, response, error, schema, and mock exist | [ ] |
| i18n | User-facing copy uses i18n keys | [ ] |
| A11y | Accessibility baseline is covered | [ ] |
| Performance | Loading, retry, pagination, and duplicate submit are handled | [ ] |
| Security | Sensitive data, permission, funds, and audit rules are covered | [ ] |
| Handoff | Page files and handoff docs are complete | [ ] |
| Runtime | Frontend mapping is implementable | [ ] |
