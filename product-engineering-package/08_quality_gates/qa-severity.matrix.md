# QA Severity Matrix

| Severity | Meaning | Release Impact |
|---|---|---|
| Blocker | User funds, permissions, audit, idempotency, or demo/live boundary can fail. | Block release. |
| Critical | KYC, ownership, limits, FX, fee, review, or account status can be incorrect. | Block production release. |
| Major | Recoverability, status clarity, method availability, or admin workflow is incomplete. | Conditional only with owner approval. |
| Minor | Copy, metadata, or non-risk polish issue. | Can defer with ticket. |
